# 拼音练习 Android WebView App 需求文档（V1）

## 1. 项目目标

将现有的「拼音练习」GitHub Pages 网站封装为 Android App，生成可安装的 APK。V1 采用 **在线 WebView 壳**：应用启动后在全屏 WebView 中加载已发布的网站，不重写既有 React 页面、练习逻辑或音频资源。

线上地址：`https://splashcurry.github.io/pinyin-practice/`

## 2. 现有网站事实

- 前端：React 19、Vite 7、TypeScript、React Router、Tailwind CSS。
- 内容：单个拼音、四声、拼读、课程、每日练习、历史练习和随机练习。
- 资源：页面通过 JavaScript 和浏览器本地存储工作，并使用静态 MP3 音频。
- 既有产品边界：没有登录、账户、后端、数据库、广告、支付、分析或学生数据采集。

## 3. 版本范围

### 3.1 V1 必须实现

1. 应用名称显示为「拼音练习」。
2. 启动后加载指定 HTTPS 网站，界面不显示浏览器地址栏。
3. 允许网站 JavaScript、DOM Storage 和媒体播放，保证练习交互、进度存储与音频可用。
4. Android 返回键：网页有历史记录时返回上一页；否则按系统默认行为退出应用。
5. 网页加载失败时显示可理解的错误页，并提供“重新加载”入口。
6. 外部链接不在应用内任意跳转：仅允许 `splashcurry.github.io` 的 HTTPS 页面继续在 WebView 打开；电话、邮件、应用商店及其他网站交给系统应用处理或明确拦截。
7. 配置网络权限；不申请与本功能无关的权限。
8. 生成可安装的 debug APK；验收通过后再生成由开发者自持密钥签名的 release APK。

### 3.2 V1 不做

- 把网站文件、音频资源打进 APK 的离线模式。
- 登录、账号、云端同步、推送、广告、支付与统计。
- 录音、语音识别、相机、定位、通讯录、存储等设备权限。
- 重新设计或重写现有网页。
- 应用商店发布材料（隐私政策页面、商店截图、AAB 上架流程）。

## 4. 使用流程

```text
用户点击“拼音练习”图标
        ↓
Android App 打开 WebView
        ↓
加载 GitHub Pages 网站
        ↓
用户完成原有拼音练习、播放音频、切换页面
        ↓
按返回键：有网页历史则返回；没有则退出
```

## 5. 非功能要求

| 项目 | 要求 |
| --- | --- |
| 网络 | V1 首次及正常使用需要网络；无网络时须能告知用户并可重试。 |
| 兼容性 | 最低 Android 8.0（API 26），以常见手机竖屏为重点，保留系统旋转能力。 |
| 隐私 | 不新增数据收集；不引入第三方广告或分析 SDK。 |
| 安全 | 仅加载 HTTPS；限制应用内导航域名；关闭不需要的文件访问能力。 |
| 可维护性 | 网站 URL、允许域名、应用版本和名称集中配置；网页更新后无需重新打 APK。 |

## 6. 验收标准

1. 安装后主屏能看到「拼音练习」图标和名称。
2. 有网络时，首次打开能进入首页；单个拼音、四声、拼读、课程、每日、历史、随机练习均可进入。
3. 各练习页面的播放按钮能正常播放音频。
4. 网页内跳转后，返回键先回到上一个网页页面，而不是直接退出。
5. 断网或临时无法访问站点时，页面出现“加载失败 / 重新加载”，应用不白屏、不崩溃。
6. `http://`、未知域名和危险 scheme 不会被静默加载进 WebView。
7. `assembleDebug` 构建成功，并能在真机安装运行。
8. release APK 经签名后可正常安装；签名密钥不提交到 Git 仓库。

## 7. 待确认的后续方向

- V1 验收后是否制作离线版。离线版需要先调整 Vite 的资源路径和路由策略，再将 `dist` 内容复制到 Android assets；它与在线版应作为 V2 单独评估。
- 是否准备自有应用图标（建议提供 512 × 512 PNG）；未提供前用 Android Studio 默认图标完成功能验证。
- 是否计划上架应用商店。若计划上架，需另行准备隐私政策、开发者主体和商店图文资料。
